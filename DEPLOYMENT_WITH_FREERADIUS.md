# FNAC + FreeRADIUS Deployment Guide

## Architecture Overview

```
Network Devices (Switches, APs)
         │
         ▼ (RADIUS requests on UDP 1812)
    FreeRADIUS Server
         │
         ├─ Reads: clients.conf (devices)
         ├─ Reads: mab_users (MAC addresses + policies)
         └─ Logs: Authentication events
         
         ▲
         │ (Config updates)
         │
    FNAC Management UI (HTTP 5000)
         │
         ├─ Device Management
         ├─ Client Management
         ├─ Policy Management
         └─ Log Viewer
```

## Prerequisites

- Astra Linux 1.7.5 or similar
- Python 3.7+
- Root/sudo access
- Network connectivity

## Step 1: Install FreeRADIUS

```bash
sudo apt-get update
sudo apt-get install freeradius freeradius-utils
```

## Step 2: Configure FreeRADIUS for MAB

Follow the guide in `FREERADIUS_MAB_SETUP.md`:

```bash
# Create initial mab_users file
sudo touch /etc/freeradius/3.0/mab_users
sudo chmod 644 /etc/freeradius/3.0/mab_users

# Test FreeRADIUS configuration
sudo freeradius -C

# Start FreeRADIUS
sudo systemctl start freeradius
sudo systemctl enable freeradius
```

## Step 3: Deploy FNAC

```bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/fnac.git
cd fnac

# Install dependencies
./quickstart.sh setup

# Start FNAC (with FreeRADIUS integration)
sudo -E ./quickstart.sh run
```

## Step 4: Access FNAC UI

Open your browser and navigate to:
```
http://server-ip:5000
```

## Step 5: Configure Devices

1. Go to **Devices** tab
2. Create a Device Group (e.g., "Switches")
3. Add a device:
   - **Device ID**: switch_1
   - **IP Address**: 192.168.1.1 (your switch IP)
   - **Shared Secret**: your_secret_here
   - **Device Group**: Switches

FNAC automatically updates FreeRADIUS `clients.conf`

## Step 6: Configure Clients (MAC Addresses)

1. Go to **Clients** tab
2. Create a Client Group (e.g., "Printers")
3. Add a client:
   - **MAC Address**: 00:11:22:33:44:55
   - **Client Group**: Printers

FNAC automatically updates FreeRADIUS `mab_users`

## Step 7: Create Policies

1. Go to **Policies** tab
2. Create a policy:
   - **Policy ID**: printer_policy
   - **Client Group**: Printers
   - **Decision**: Accept with VLAN
   - **VLAN ID**: 100

FNAC automatically updates FreeRADIUS with VLAN assignments

## Step 8: Test Authentication

### From the server:
```bash
# Test with radtest
radtest -t mschap 00:11:22:33:44:55 mab 127.0.0.1 1812 your_secret_here

# Expected response:
# Received Access-Accept Id 123 from 127.0.0.1:1812
```

### From a network device:
Configure your switch/AP to send RADIUS requests to the FNAC server:
- RADIUS Server: server-ip
- RADIUS Port: 1812
- Shared Secret: your_secret_here

## Configuration Files

### FreeRADIUS Config (auto-generated by FNAC)

**`/etc/freeradius/3.0/clients.conf`**
```
# Auto-generated by FNAC
client 192.168.1.1 {
    secret = "your_secret_here"
    shortname = "switch_1"
    nastype = generic
}
```

**`/etc/freeradius/3.0/mab_users`**
```
# Auto-generated by FNAC
00:11:22:33:44:55 Cleartext-Password := "mab"
    Tunnel-Type = VLAN,
    Tunnel-Medium-Type = IEEE-802,
    Tunnel-Private-Group-ID = "100"
```

## Troubleshooting

### FreeRADIUS not responding

1. Check if FreeRADIUS is running:
   ```bash
   sudo systemctl status freeradius
   ```

2. Check for errors:
   ```bash
   sudo freeradius -X
   ```

3. Verify configuration:
   ```bash
   sudo freeradius -C
   ```

### Clients not authenticating

1. Verify MAC address format (XX:XX:XX:XX:XX:XX)
2. Check MAC is in FNAC Clients list
3. Check policy is created for the client's group
4. Verify shared secret matches in device configuration

### VLAN not assigned

1. Check policy decision is "Accept with VLAN"
2. Verify VLAN ID is valid (1-4094)
3. Check network device supports VLAN assignment
4. Verify FreeRADIUS config was updated:
   ```bash
   sudo grep -A 3 "00:11:22:33:44:55" /etc/freeradius/3.0/mab_users
   ```

### Config not updating

1. Check FNAC is running with sudo:
   ```bash
   sudo -E ./quickstart.sh run
   ```

2. Check permissions on FreeRADIUS config files:
   ```bash
   ls -la /etc/freeradius/3.0/clients.conf
   ls -la /etc/freeradius/3.0/mab_users
   ```

3. Check FNAC logs for errors
